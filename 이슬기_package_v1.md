**현재 처리 방식:**

- 담당자가 반복 문의 응대로 인해 본인의 핵심 업무에 집중하기 어려운 상황

# 📋 개인 맞춤 바이브코딩 실습 계획

## 🎯 최종 목표
**mySUNI 총무 업무 자동 응답 챗봇 만들기**
- 임시출입증, 주차등록 등 반복 문의에 자동 답변
- 웹 인터페이스에서 질문 입력 → AI가 답변 생성
- 실제 업무에 바로 활용 가능한 최소 동작 서비스

---

### 1단계: 총무 FAQ 데이터 준비 및 기본 챗봇 화면 만들기

**요약:**

• Google Sheets에 총무 업무 FAQ 정리 (질문-답변 쌍 10~15개)
  - 예: "임시출입증 발급 방법은?", "주차등록은 어떻게 하나요?" 등
  - Cursor agent에게 "Google Sheets 연동용 Python 코드 작성해줘" 요청

• Streamlit으로 기본 챗봇 화면 구성
  - Cursor agent 모드에서 "Streamlit으로 채팅 UI 만들어줘" 프롬프트 입력
  - 사용자가 질문 입력 → 화면에 표시되는 기본 구조 생성

• 로컬에서 `streamlit run app.py` 실행하여 화면 확인
  - Cursor 터미널에서 직접 실행, 브라우저에서 http://localhost:8501 접속

**기대 결과:**  
웹 브라우저에서 질문을 입력할 수 있는 챗봇 화면이 뜨고, Google Sheets에 정리한 FAQ 데이터를 Python 코드로 불러올 수 있는 상태

---

### 2단계: Gemini API 연결 및 자동 답변 생성 구현

**요약:**

• Gemini API 키 발급 (Google AI Studio에서 무료 발급)
  - Cursor agent에게 "Gemini API 연동 코드 작성해줘" 요청
  - `.env` 파일에 API 키 저장하는 방법 안내받기

• FAQ 데이터를 Gemini에게 "참고 자료"로 전달
  - Cursor agent 모드: "Google Sheets 데이터를 Gemini 프롬프트에 포함시켜줘"
  - 사용자 질문 + FAQ 내용을 함께 보내서 정확한 답변 생성

• 챗봇 화면에서 질문 입력 → Gemini 답변 표시
  - "임시출입증 발급 방법은?" 입력 시 → FAQ 기반 자동 답변 출력
  - Cursor agent에게 "답변을 채팅 형식으로 예쁘게 표시해줘" 요청

**기대 결과:**  
실제 총무 업무 질문을 입력하면 Google Sheets FAQ를 참고하여 Gemini가 자동으로 답변을 생성하고, 웹 화면에 즉시 표시됨

---

### 3단계: 답변 품질 개선 및 문의 이력 기록

**요약:**

• 답변 정확도 높이기 위한 프롬프트 개선
  - Cursor agent에게 "총무 담당자 역할로 친절하게 답변하도록 프롬프트 수정해줘" 요청
  - "모르는 질문은 '담당자에게 문의하세요'라고 안내하도록 설정해줘"

• 문의 내역을 Google Sheets에 자동 저장
  - Cursor agent 모드: "질문과 답변을 Google Sheets 새 시트에 기록하는 코드 추가해줘"
  - 날짜, 질문, 답변, 만족도(옵션) 컬럼 구성

• 간단한 피드백 버튼 추가 (👍/👎)
  - Cursor agent: "답변 아래 좋아요/싫어요 버튼 추가하고 결과를 Sheets에 저장해줘"
  - 나중에 어떤 질문이 많은지, 답변 품질은 어떤지 분석 가능

**기대 결과:**  
챗봇이 총무 담당자처럼 친절하고 정확하게 답변하며, 모든 문의 내역이 Google Sheets에 자동 기록되어 나중에 분석하거나 FAQ 업데이트에 활용 가능

---

### 4단계: 실제 업무 환경에서 테스트 및 개선

**요약:**

• 동료 또는 팀원에게 챗봇 공유하여 실제 질문 테스트
  - Cursor agent: "Streamlit 앱을 로컬 네트워크에서 접속 가능하게 설정해줘"
  - 같은 Wi-Fi 환경에서 다른 PC로 접속 테스트 (예: http://192.168.x.x:8501)

• 자주 묻는 질문 중 답변이 부족한 부분 FAQ에 추가
  - Google Sheets에 새로운 질문-답변 추가 → 즉시 챗봇에 반영됨
  - Cursor agent: "Sheets 데이터 실시간 새로고침 기능 추가해줘"

• 보안 고려사항 체크
  - API 키가 코드에 노출되지 않도록 `.env` 파일 사용 확인
  - 민감한 정보(개인정보 등)는 FAQ에 포함하지 않도록 주의

**기대 결과:**  
실제 mySUNI 총무 업무에서 발생하는 질문에 즉시 답변할 수 있는 챗봇이 완성되고, 동료들이 직접 사용해보며 피드백을 받아 지속적으로 개선 가능한 상태

## ✅ 단계별 체크리스트

### 1단계: 핵심 환경 빠른 검증 (5분)

- [ ] Cursor 에디터 하단 터미널에서 `python --version` 입력 후 Python 3.8 이상 확인 — 실패 시: 01 환경설정 가이드로 돌아가서 Python 재설치
- [ ] 터미널에서 `streamlit --version` 입력하여 Streamlit 설치 확인 — 실패 시: `pip install streamlit` 명령어로 설치
- [ ] 프로젝트 폴더에 `.env` 파일이 있고, 안에 `GEMINI_API_KEY=AI...` 형태로 키가 저장되어 있는지 확인 — 실패 시: 01 가이드 "API 키 발급" 섹션 다시 진행
- [ ] API 키가 39자 길이이고 "AI"로 시작하는지 확인 (예: AIzaSyA...) — 실패 시: Google AI Studio에서 키 재발급 후 .env 파일에 다시 붙여넣기

---

### 2단계: 총무 챗봇 프로젝트 폴더 구조 만들기 (3분)

- [ ] 프로젝트 폴더 안에 `app.py` 파일 생성 (메인 챗봇 코드가 들어갈 파일) — 실패 시: Cursor에서 "New File" 클릭 후 이름 입력
- [ ] 같은 폴더에 `requirements.txt` 파일 생성 (필요한 패키지 목록 저장용) — 실패 시: 메모장으로 만들어서 확장자를 .txt로 저장
- [ ] Google Sheets에 "총무FAQ" 시트 만들고 첫 줄에 "질문", "답변" 컬럼 추가 — 실패 시: 새 스프레드시트 생성 후 A1에 "질문", B1에 "답변" 입력
- [ ] FAQ 시트에 테스트용 질문 3개 입력 (예: "임시출입증 발급 방법은?", "주차등록 절차는?", "사무실 위치는?") — 실패 시: 간단한 질문 1개라도 입력하면 됨
- [ ] 프로젝트 폴더를 `mySUNI_chatbot` 같은 알기 쉬운 이름으로 변경 — 실패 시: 폴더 우클릭 → 이름 바꾸기

---

### 3단계: 첫 구현 준비 - Cursor Agent 활용 (5분)

- [ ] Cursor에서 `Ctrl+L` (또는 Cmd+L) 눌러 Chat 창 열기 — 실패 시: 상단 메뉴에서 "View" → "Chat" 클릭
- [ ] Chat 창에 다음 프롬프트 입력: **"Streamlit으로 간단한 채팅 UI 만들어줘. 사용자가 질문 입력하면 화면에 표시되는 기본 구조"** — 실패 시: "Streamlit 채팅 예제 코드 작성해줘"로 더 간단하게 요청
- [ ] Agent가 생성한 코드를 `app.py`에 복사 붙여넣기 (Cursor가 자동으로 해줄 수도 있음) — 실패 시: 코드 블록 전체 선택 후 Ctrl+C, app.py에 Ctrl+V
- [ ] 터미널에서 `streamlit run app.py` 입력하여 웹 화면 실행 — 실패 시: 에러 메시지를 Cursor Chat에 붙여넣고 "이 에러 해결해줘" 요청
- [ ] 브라우저에서 `http://localhost:8501` 자동으로 열리고 입력창 보이는지 확인 — 실패 시: 브라우저 주소창에 직접 입력해서 접속
- [ ] 입력창에 "테스트" 입력 후 화면에 표시되는지 확인 (답변 기능은 아직 없어도 됨) — 실패 시: 정상 작동, 다음 단계에서 답변 기능 추가 예정

---

### 4단계: Google Sheets 연결 준비 (5분)

- [ ] Google Cloud Console에서 "Google Sheets API" 검색 후 사용 설정 클릭 — 실패 시: 01 환경설정 가이드 "Google Sheets 연동" 섹션 참고
- [ ] "사용자 인증 정보" 메뉴에서 "서비스 계정" 만들기 (이름: mySUNI-chatbot) — 실패 시: "서비스 계정 만들기" 버튼 찾아서 클릭 후 이름만 입력
- [ ] 서비스 계정 생성 후 "키 추가" → "JSON" 선택하여 키 파일 다운로드 — 실패 시: 서비스 계정 목록에서 방금 만든 계정 클릭 → "키" 탭 이동
- [ ] 다운로드한 JSON 파일을 프로젝트 폴더에 `service_account.json` 이름으로 저장 — 실패 시: 파일 이름 바꾸기로 변경
